  - `Content` was speced as one document *per block*, chained via `prevContent`/`nextContent`. The linked-list design would have meant constantly translating between that shape and Blocknote's own on every keystroke, for a "block-independence" property Blocknote already gives you natively. Replaced with one `Content` document per `Page` holding the whole array.

- Two TypeScript inference bugs, both fixed before the lint/typecheck pass shipped clean, both from the same root cause (an inferred, not declared, function return type leaking a wider union than intended):
